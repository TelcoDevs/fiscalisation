ZIMRA Fiscalisation

Complete Fiscal Device Integration for ZIMRA Compliance

Overview

The ZIMRA Fiscalisation module provides a comprehensive solution for businesses in Zimbabwe to comply with the Zimbabwe Revenue Authority (ZIMRA) fiscal requirements. This module integrates your Odoo accounting system with approved fiscal devices, ensuring all transactions are properly reported to tax authorities.

Important: This module requires a compatible ZIMRA-approved fiscal device and internet connectivity to the FDMS (Fiscal Device Management System) server.

Key Features

Automated Token Management

Automatic handling of authentication tokens with the fiscal server, including scheduled refreshing to maintain continuous connectivity.

Fiscal Day Management

Open and close fiscal days with proper tracking of fiscal day numbers, receipts, and status.

Invoice Fiscalisation

Seamless fiscalisation of customer invoices with proper tax calculations, buyer data submission, and HS code handling.

QR Code Generation

Automatic generation of QR codes for fiscalised invoices, providing easy verification of fiscal compliance.

Comprehensive Error Handling

Detailed error tracking and notification system to quickly identify and resolve integration issues.

Credit Note Support

Full support for credit notes and refund processes with proper fiscal reporting.

Configuration Guide

Step 1: Fiscal Device Setup

  1. Navigate to Accounting → Configuration → Fiscal Devices
  2. Click Create to add a new fiscal device
  3. Enter the device name, device ID, and serial number
  4. Input the activation key provided by your fiscal service provider
  5. Verify the API base URL (usually https://fiscal-demo.telco.co.zw for testing or production https://fiscal.telco.co.zw for live environment)
  6. Save the configuration

Step 2: Configure Products

  1. Ensure all products have the correct HS Code set
  2. Verify that appropriate taxes are configured for each product
  3. Make sure all product prices are correctly configured with tax inclusion settings

Step 3: Configure Customer Information

  1. Ensure all customers have complete information including:
    • VAT Number (if applicable)
    • TIN Number (if applicable)
    • Complete address information
    • Contact details (phone, email)

Daily Operations

Fiscalising Invoices

To fiscalise an invoice:

  1. Create and validate an invoice as usual in Odoo
  2. Open the validated invoice
  3. Click the Fiscalise button
  4. The system will communicate with the fiscal device and update the invoice with fiscal information
  5. A QR code will be generated and displayed on the printed invoice

Closing a Fiscal Day

At the end of business day:

  1. Navigate to Fiscalisation → Fiscal Devices
  2. Select your fiscal device
  3. Click the Close Day button

Troubleshooting

Common Issues

Issue Possible Solution
Token Refresh Failure Check internet connection and device activation details. Use the manual token refresh button.
Device Not Found Verify the device serial and activation key. Ensure the device is registered with ZIMRA.
Missing HS Code Error Update product configuration with the correct HS Code.
Mixed Tax Inclusion Types Ensure all invoice lines have consistent tax inclusion settings.
Duplicate Invoice Number If you see error RCPT013, the invoice was likely already fiscalised.
